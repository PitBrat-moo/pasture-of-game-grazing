Revenant
========
  
  
NOTES  
-----  
- Run Launcher.exe and set detail levels every time before starting the game. WARNING, opening Launcher.exe will reset configuration to defaults
- Play the game with Launcher.exe. WARNING, opening Revenant.exe can make the game appear too bright
- Cap fps at 30 to match the internal game engine for stability **required**

VM CONFIGURATION
---------------------------
• CPU Frequency: 300 MHz  
• CD-ROM Speed: 72x 
• Renderer: OpenGL
• Target Framerate: 60 fps **favor stability**
• Win98SE  
• Desktop Resolution: 800x600  
• Desktop Color Depth: 16‑bit (High Color)  

VM INSTALLATION 
---------------------------
• Set CPU Frequency to <=300MHZ **favor stability**
• Start VM
• Mount Revenant disc 1
• Install Revenant inside the VM
  → Complete DirectX 6.1 installation
• Install Patch 1.22
  → Select game folder → Overwrite → Yes to All
• Install NoCD patch
• Install Dialogue Fix
  → copy Ahkuilon.rvm into the “modules” folder
• Run Launcher.exe once to configure settings and generate Revenant.ini
  → Run Revenant → Display Device → Primary display adapter
  → Display Mode: Full Screen
  → Advanced Options
    • Force Software Rendering: Disabled **favor stability**
    • Disable Fast Surface Locks: Disabled **favor beauty**
    • Preload Sectors: Enabled
    • Preload Sector Size: Maximum
    • Chunkcache Size: Maximum
    • Detail Level: Maximum
  → Start Game → Revenant.ini automatically generated
    • Game Options → Enable Real Time Lights: Enabled (re-enable each game session)
    • Launcher.exe always resets settings; configure every time before starting game

RTSS CONFIGURATION (strongly recommended)
---------------------------
• 86Box.exe → Framerate limit: 30 **critical for stable timing** 

SHADERGLASS CONFIGURATION (OPTIONAL)
---------------------------
• Crop Input Top: 22 **removes vm menu bar**  
• Pixel Size: x1
• Custom Shader: Bovinated Revenant Shader

SRWE CONFIGURATION (OPTIONAL)  
---------------------------
• y: -31 **hide Shaderglass menu bar**
• Width: 2896 **add 16 pixels to compensate for window frame at 4x3 resolution**  
• Height: 2199 **add 39 pixels to compensate for window frame at 4x3 resolution**  
• Align Windows: H.Center

LOSSLESS SCALING SETTINGS (OPTIONAL)
---------------------------
• Crop Input → Top Box: 22 **crops 86Box menu bar**  
• Frame Generations Type: LSFG 3.1  
• Mode: Fixed
• Multiplier: 7.8 (235 fps)
• Flow Scale: 30%  
• Performance: ON **preserve VGA beauty**  
• Capture API: WGC  
• Queue target: 0 **1 for compatibility**
• Clip cursor: On **Optional**
• Adjust cursor speed: On **Optional**
• Scaling Type: Sharp Bilinear **preserve VGA beauty**  
• Sync Mode: Vsync
• Max frame latency: 1
• G-Sync support: ON

<>>------------------><<
STARTUP ROUTINE
>><------------------<<>
• Windows-Host → Select Sound Playback Device → Attach Controller → Launch SRWE → Launch Shaderglass → Launch RTSS
• Launch 86Box → Sart Win98se VM → Mount Revenant Disk 2
• Shaderglass: Select 86Box as Input Window; 
• SRWE: Select Shaderglass and set size/position
• Win98SE-Guest: Launch Launcher.exe → Configure cache and detail  

Notes:
- VM path avoids modern rendering issues
- VM path is the only path that enables redbook CD-Audio
- Game timing is stable under Win98SE with host-side 30 fps cap
- In 86Box, set CPU speed to 300 to avoid audio jitter and the training bug
  • 86Box → Settings → Processor → Frequency: 300 **233 for compatibility**
- Requires host-side RTSS 30 fps limit; launch RTSS before 86Box  
- Use S3 with software rendering for best gameplay
  • Voodoo3 DirectX hardware mode is unstable but the game is completable and visually authentic; its software path is broken
  • Voodoo1/2 Glide is incomplete and non-functional

TWEAKS & FIXES
--------------
Settings Reset on Startup  
• Configure settings every time before starting the game 
• Do not run the game from Revenant.exe; it breaks gamma and makes the game look too bright  
• Run launcher and configure settings every time

The game is too dark or bright
• Pressing OK in the menu can make the game too bright or dark; Restart game client; do not change settings in-game

Game Crashes  
• Disable “Fast Surface Locks” in launcher advanced options

Visual effects stay active / minimap corruption / animation jitter
• Requires 30 fps cap; save and reload to clear effect

Character keeps walking
• Open menu to clear input state

CD‑Audio bug: after loading a new area, CD volume increases to 100%  
• Open Menu (press ESC) → Game Options; volume returns to normal  

Do not force software renderer; it makes the game unstable. Allow the engine to automatically choose the rendering path for stability 
• Revenant.ini → Software3D=No  

Install disc 2 movies to the game folder
• Copy contents of disc 2 into a new folder inside the games folder: .\Disc2
  → Open Revenant.ini and change:
    • MoviePath=".\Disc2"

Spell effects cause engine slowdown (geyser + flame + buffs + melee)
• Lower Detail Level by one notch to reduce particle/event load with no noticeable visual loss

KNOWN ISSUES (UNRESOLVED)
-------------------------
A4R4G4B4 Texture Failure (DXWND‑CONFIRMED)  
• Dialog text, UI overlays, minimap elements fail to render  
• Engine-level issue; no known fix

Lighting / Gamma Problems  
• Black areas at low gamma  
• Interiors darken randomly  
• Candle flames show black rectangles  
• Gamma slider can exceed intended bounds

Chroma‑Key Artifacts (on modern Windows)  
• Magenta outlines around text and UI elements

Revenant cannot survive losing focus; it will crash

The game freezes for a few milliseconds during loot spawns when using DirectX hardware mode
