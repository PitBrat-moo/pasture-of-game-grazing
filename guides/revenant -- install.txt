Revenant.txt
============

INSTALLATION
------------

A. VM INSTALLATION (Recommended)
• Create a Windows 98 SE virtual machine in 86Box
• Install Revenant inside the VM
• Install Patch 1.22
• Install NoCD patch
• Install Dialogue Fix (copy Ahkuilon.rvm into the “modules” folder)

Startup Routine:
- Windows-Host → Select Sound Playback Device → Attach Controller → SRWE → Shaderglass → 86Box → Win98se vm → Mount Revenant Disk 2 → Shaderglass: Select 86Box as Input Window → SWRE: Select Shaderglass and set size/position → Win98se: Launch Revenant.exe  

Notes:
- VM path avoids modern rendering issues
- Game timing is stable under Win98SE
- In 86Box, set CPU speed to 300 to avoid audio jitter and a bug that prevents completing training.
  • 86Box → Settings → Processor → Frequency: 300 **233 for compatibility**

B. WINDOWS 10/11 INSTALLATION
-----------------------------
Major issue: magenta chroma‑key artifacts around text (no known fix)

Steps:
• Install the game normally
• Install dgVoodoo (latest version unless otherwise specified)

dgVoodoo Settings:
  General → Windows
    - Scaling: Unspecified
  DirectX
    - Uncheck “Watermark”
    - Uncheck “Application controlled fullscreen”
    - Check “Disable Alt‑Enter”
  GeneralExt
    - FPS Limit: 30

Notes:
- FPS limit is mandatory
- Chroma‑key artifacts remain unresolved on modern systems


NOTES  
-----  
- Start the game with Launcher.exe at least once to configure settings  
- Launch the game with Revenant.exe, using Launcher.exe resets configuration data to default values (Keep a backup of revenant.ini)

TWEAKS & FIXES
--------------

Settings Reset on Startup  
• Run the game from Revenant.exe instead of Launcher.exe  
• Must run launcher once to configure settings  
• Game speed resets every session

Game Crashes  
• Disable “Fast Surface Locks” in launcher advanced options

Game Fails to Boot / Large Black Areas  
• Limit RAM access to 2 GB  [unverified]

Intro Movie Freeze  
• Skip by removing / renaming SMK files or using wrappers with “no movies” flags


KNOWN ISSUES (UNRESOLVED)
-------------------------

A4R4G4B4 Texture Failure (DXWND‑CONFIRMED)  
• Dialog text, UI overlays, minimap elements fail to render  
• Engine-level issue; no known fix

Lighting / Gamma Problems  
• Black areas at low gamma  
• Interiors darken randomly  
• Candle flames show black rectangles  
• Gamma slider can exceed intended bounds

Chroma‑Key Artifacts  
• Magenta outlines around text and UI elements


VERIFIED FIXES
--------------
• Disable “Fast Surface Locks”


UNVERIFIED / HISTORICAL FIXES
-----------------------------
• Disable Z Buffering  
• Forced Software Rendering  
• Software3D=Yes  
• Limit RAM to 2GB  
• Disable texture blitting / overlays  
• Software renderer for minimap (severe performance loss)
