Windows 10/11 Installation
------------------------------

Install game  
If installing from disc, install Patch 1.7b  
   • Two_Worlds_Update_1.0-1.7b.exe  

Right‑click TWOWORLDS.EXE → Properties → Compatibility  
   • Disable fullscreen optimizations: Enabled  
   • Change high DPI settings  
       → Override high DPI scaling behavior: Enabled  
       → Set dropdown to Application  
   • Compatibility Mode: Windows Vista  

Cap framerate to 60 in NVIDIA Control Panel (ignored by game, but safe)  

Launch game  

If installing from disc, activation is required  
   • Enter Serial  
       → Use copy/paste if the text box does not accept typed input  
   • Activate Online  
   • Copy/paste information into form  
   • Registration will ask about connection  
   • Click Continue  
   • Connection will fail and the game will automatically activate  

Use RTSS to set the framerate limit to 60  
   Notes:  
   • Animation and physics break above 60 FPS  
   • The game ignores driver‑level framerate caps

----------------

   Notes:

   • Autosave causes the game to crash

Configure AutoexecGame.con  
--------------------------  
Notes:  
   • The game does not automatically generate a configuration file  
   • These commands also work from inside the game console  
   • Setting Engine.FOV > 50 breaks the game engine  

Navigate to <path‑to‑game>  
If it doesn't already exist, create a folder named “Parameters”  
Inside Parameters, create a text file named “AutoexecGame.con”  
Open AutoexecGame.con in a text editor, add one of the following blocks, and save

LOSSLESS SCALING  
----------------  
Notes:  
   • Use RTSS to cap framerate to 60 for frame generation  
   • Requires Window Mode  
     → Launch game → Settings → Graphics → Window Mode: Enabled  

• Frame Generations Type: LSFG 3.1  
• Mode: Fixed  
• Multiplier (base cap 60 FPS):  
     → 3.9  (240 Hz @ ~234 FPS)  
     → 1.9  (120 Hz @ ~114 FPS)  
• Flow Scale: 30%  
• Performance: ON  
• Capture API: DXGI (WGC for compatibility)  
• Queue target: 0 (1 for compatibility)  
• Clip cursor: Off (optional)  
• Adjust cursor speed: Off (optional)  
• Scaling Type: Sharp Bilinear  
• Rendering Sync Mode: Default  
• Max frame latency: 2  
• G‑Sync support: ON

TWEAKS & FIXES  
--------------  
Enable/Disable intro videos  
   Registry:  
      HKEY_CURRENT_USER\Software\Reality Pump\TwoWorlds\Interface  
      ShowIntro: 0  (1 = Enabled, 0 = Disabled)  

UI Scale Adjustment  
   Note: Values other than 0 (autoscaling) may cause corruption or crashes  
   Registry:  
      HKEY_CURRENT_USER\Software\Reality Pump\TwoWorlds\Interface  
      ForceScaleX / ForceScaleY  
      Set Base: Decimal  
      Start low (e.g., 640 × 480) and adjust upward  
    
Console Commands  
   • Display.Show 1  (Show stats HUD; default 0)  
  
The game crashes immediately on AMD GPUs  
   • Install the game on the C: drive  
  
Typing doesn’t work in the serial box or console  
   • The text input widget is broken on modern Windows  
   • Use copy/paste to enter text  

Missing DX9 components (xinput1_3.dll, d3dx9_31.dll)
   • Install DirectX June 2010 Runtime   

Missing voices or music
   • Note: Two Worlds uses Windows Media Audio/Video 9 codecs
   • Install K‑Lite Codec Pack (Standard or Mega)

NPC stutter / freeze when many footsteps play
   • Set Windows audio format to 16‑bit, 44.1 kHz
   • Avoid 7.1/Atmos; use Stereo, 4.0, or 5.1

Disable intro videos to prevent ReShade startup freezes
   • Navigate to <path-to-game>\Data\Video
   • Delete or rename these files:
       Intro2.wmv
       Intro3.wmv
       Intro5.wmv
   • Note: Some editions use different filenames (e.g., Intro.wmv, RealityPump.wmv, TopWare.wmv)

The game crashes after ~3-6 manual saves
   • Reload immediately after saving — it resets the engine state
   • In towns with lots of NPCs or near water, restart the game every 2–3 saves  
   • Lowering the resolution to ≤ 2560x1440 can improve save stability

KNOWN ISSUES   
------------  
• HUD cannot be centered on ultrawide resolutions (hard‑coded)  
• MSAA works only in fullscreen mode (engine limitation) 
• Text input fields may not accept typed characters — use copy/paste  
• FOV above 50 breaks geometry and lighting  
• Autosave can cause crashes (disable it) 
• Quicksave can cause crashes (avoid it) 
• With ReShade installed, the game may occasionally freeze on a black screen before the intro video
  This is a ReShade hook‑timing issue. Gameplay stability is not affected.
