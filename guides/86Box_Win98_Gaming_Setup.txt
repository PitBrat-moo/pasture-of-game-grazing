⎯(ᶜᵒʷ•)⎯─◐◑◒◓─ MACHINE

⎯(ᶜᵒʷᵔ)⎯─◐◑◒◓─ ENTER BIOS, SET RENDERER AND PREPARE WINDOWS INSTALLATION  
→ Enter BIOS: Start VM → Press DEL during boot
→ Set Renderer: 86Box VM window → View → Renderer → OpenGL **favor quality**

  • Some older FMV/DirectDraw titles may prefer **30 fps** (optional; adjust per‑game after install).

⎯(ᶜᵒʷ•ᵔ)⎯─◐◑◒◓─ DISABLE IRQ STEERING **Avoids Poisoning**
→ Device Manager → System devices → PCI bus → IRQ Steering → Use IRQ Steering: Disabled
→ Reboot → BIOS → [BOOT] → Reset Configuration Data: Yes **force IRQ/DMA refresh**
→ Reboot → Cancel driver installation **favor caution**

⎯(ᶜᵒʷ•ᵔ)⎯─◐◑◒◓─ VOODOO 3 3000 AND HOST RENDERER
Description: Install Voodoo3 drivers.
→ (Optional) Device Manager → If “Standard PCI Graphics Adapter (VGA)” appears, remove it **favor caution**
→ Install 3dfx Voodoo3 reference driver 1.07
→ Reboot → Cancel generic display driver installation → Continue Voodoo3 Installation
→ Reboot → BIOS → [BOOT] → Reset Configuration Data: Yes **force IRQ/DMA refresh**

⎯(ᶜᵒʷ•ᵔ)⎯─◐◑◒◓─ YAMAHA YMF719 (OPL3-SA3) SETUP
Description: Install Yamaha YMF719 (OPL3-SA3) drivers with correct IRQ/DMA (5,0,1)
→ Verify IRQ Steering is disabled in Device Manager → System devices → PCI bus → IRQ Steering: Disabled **favor caution**
→ Remove Yamaha devices: Device Manager → Other Devices → MPU-401 Compatible, OPL-SAX Sound Board (both instances)
→ Install YMF719 Driver v2.3.43 for Win95 (95v2343)
→ Reboot → BIOS → [BOOT] → Reset Configuration Data: Yes **force IRQ/DMA refresh**
→ Verify IRQ for Yamaha YMF719 (OPL3-SA3) **favor caution**
  • Device Manager → System devices → Sound, video and game controllers → YAMAHA OPL3SAx → Resources
    → Interrupt Request: 5, Direct Memory Access: 0, Direct Memory Access: 1

⎯(ᶜᵒʷ•)⎯─◐◑◒◓─ VOODOO3 REG TWEAK (IMPORT)
Description: .reg file to set alpha dither mode — purpose: reduce banding and stabilize Glide alpha blending.
→ Steps:
   1. Open Notepad, paste the REG block below, save as `3dfx_voodoo3_alphadither.reg`.  
   2. Double‑click the file in guest → Accept prompts → Reboot.  
   3. Verify in RegEdit under `HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\Class\Display\0000\Glide`.  
   → **Note:** if your card appears under a different display index (e.g., `0001`), adjust the path accordingly.

--- REG BLOCK — IMPORTANT DISPLAY INDEX (READ BEFORE IMPORT) ---
; Verify the display index before importing.
; If your Voodoo3 appears under a different display node, change:
; HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\Class\Display\0000\Glide
; to the correct node (for example ...\Display\0001\Glide).

[HKEY_LOCAL_MACHINE\SOFTWARE\3dfx Interactive\Glide]

⟦⟦⟦~⟡🐄⟧⟧⟧ QUICK TROUBLESHOOTING ⟦⟦🐄⟣~⟧⟧⟧
Symptom | Likely cause | First action
⟢ FMV drops / stutter | Per‑game FMV timing sensitivity | Set MHz per‑game: FMV‑heavy titles prefer lower MHz; 3D‑heavy titles prefer higher MHz
  • Optional: lower CD‑ROM speed (24×/12×) for disc-image read consistency
⟢ Low FPS in 3D | CPU bottleneck | Use 450 MHz frequency for that title; reduce frequency if audio jitter
⟆⟆~⟡~⟆⟆   ⇢ GLIDE GRAPHICS CORRUPTION ⇠   ⟅⟅~⟣~⟅⟅  
→ If a game crashes or shows corrupted textures, delete its local glide2x.dll: Tomb Raider, Carmageddon, Fatal Racing, Screamer 2
  • Delete 'glide2x.dll' in game folder
⟦⟦~⟣~⟧⟧⟧ END QUICK TROUBLESHOOTING ⟦⟦~⟡~⟧⟧⟧

⟊⟊⟊~⟡🐮⟡~⟊⟊⟊   ⇢ FREQUENCY TUNING — THE PER‑GAME RITUAL ⇠   ⟉⟉⟉~⟣🐮⟣~⟉⟉⟉
Description: Determine optimum CPU frequency (MHz) settings on a per‑game basis.  
→ Check recommended system requirements for the title.  
  • FMV‑heavy titles: ~100–133 MHz
  • Dialogue-heavy & 3D/2D hybrid titles: ~233–300 MHz
  • 3D Accelerated titles: ~333–366 MHz
  • Late era titles (2000–2002): ~366–400 MHz
→ Run three titles:  
   • FMV / Dialogue / MIDI heavy title — listen for audio jitter and note timing.
   • Redbook / CD audio title — verify track switching and volume.
   • 3D Accelerated title (Glide, OpenGL, D3D) — note emulation speed dips, frame pacing, and jitter.  
→ If jitter, reduce MHz and retest.
→ Log results: title, tuned frequency.

⎯(ᶜᵒʷ•ᵎ)⎯─◐◑◒◓─ FINAL CHECKLIST BEFORE PLAY
→ 86Box: Voodoo3 3000 = Primary ; Yamaha YMF719 (OPL3-SA3) = **IRQ 5 / DMA 0 / HDMA 1** ; HDD 16GB `.img` Primary Master DMA 0:0 ; CD‑ROM ATAPI Secondary Master 1:0 `86b_cd 3.50` DMA 72× ; Floppy 1.44MB ; PS/2 mouse 5 buttons + wheel.  
→ 86Box: **Ports (COM & LPT)** → disable unused COM/LPT to match BIOS; re‑enable only when needed.  
→ BIOS: PNP OS = NO ; USB disabled ; Primary/Secondary IDE Slave = DISABLED unless devices attached ; Video BIOS shadow DISABLED.  
→ Windows: Voodoo3 drivers, Yamaha VxD, DirectX 7.0a installed, Shutdown Fix; Device Manager shows DMA enabled.  
→ Mouse: PS2Plus polling = 80 Hz confirmed. **favor compatibility**  
→ 3dfx Tools UI: settings as documented.  
→ Registry: `SSTH3_ALPHADITHERMODE="3"` present under `...Display\0000\Glide` (adjust index if card appears under `0001`).

⎯(ᶜᵒʷ•)⎯─◐◑◒◓─ LAUNCH SEQUENCE
Note: Enable the host audio device and joystick before launching 86Box; they will not be detected afterward.  
      Launch RTSS before 86Box; starting it afterward can cause a crash.
→ Windows Host → Select Audio Device → Attach Joystick → Launch RTSS (optional fps cap)
→ Launch VirtualMIDISynth (optional host MIDI)
→ Launch ShaderGlass or Lossless Scaling (choose one); use SRWE for scaling when using ShaderGlass
→ Launch 86Box → Start VM → Boot Win98 → Mount CD-ROM → Launch game

⎯(ᶜᵒʷ•ᵔ)⎯─◐◑◒◓─ FILE TRANSFER RITUAL
→ Place all driver and file offerings into a host‑side folder.
   • Extract archives (.zip, .rar) on host **only place extracted files in host-side folder** **86Box can not mount large, complex folders**
   • Organize files by device and you will avoid future sorrow.
   • Align the folder structure to an actual CD-ROM **shallow directories and 300-500 total files and folders combined is safe**
→ Boot the Win98se guest and mount the host-side folder as a CD‑ROM.
   • 86Box vm window → Media → CD‑ROM → Folder → Select folder: host-side folder
→ Guest → Open “My Computer”.
   • The folder will appear as a CD‑ROM volume.
→ Copy all necessary files from the mounted host-side folder CD-ROM to the guest HDD.
   • Copy into a safe place inside the guest → C:\DRIVERS\
→ Only after the files graze safely inside the guest pasture should you begin installing them.

⚠ W A R N I N G ⚠  
Do not install drivers directly from the mounted folder.  
Copy first, install second. This preserves Stability and prevents poisoning.

⎯(ᶜᵒʷ•ᵔ)⎯─◐◑◒◓─ THE POISONED INSTALL
Description: The system installs WDM instead of VxD drivers. Devices are locked to the wrong IRQ/DMA and stop functioning. Removing the device and reinstalling drivers does not restore functionality. 
→ Restore the image backup created before restarting after Win98 setup.
→ Reboot → BIOS → [BOOT] → Reset Configuration Data: Yes **force IRQ/DMA refresh**
→ Boot Windows → Cancel driver installation → Continue with Windows setup

⎯(ᶜᵒʷ•ᵔ)⎯─◐◑◒◓─ LOSSLESS SCALING
Description: Lossless Scaling settings optimized for VGA gaming at 60Hz **not compatible with Shaderglass**
→ Frame Generation → Type: LSFG 3.1 → Mode: Fixed → Multiplier: 4 → Flow scale: 100% → Performance: ON **preserves VGA detail**
→ Capture → Capture API: WGC; Queue target: 0
→ Scaling → Type: Sharp Bilinear **preserves VGA detail**
→ Rendering → Sync mode: Off; Max frame latency: 1; G-Sync support: On
→ Crop Input → Top: 22; Right: 0; Bottom: 0; Left: 0 **crop vm menu bar from output**
→ 86Box vm window → View → Hide toolbar: ON; Hide status bar: ON **hide toolbar from scaled output**
→ 86Box vm window → View → Renderer options → Use target framerate: 60 fps
   • Alternately, use RTSS to cap fps. **NVIDIA/AMD frame limiter does not work on vm window**

⎯(ᶜᵒʷ•ᵔ)⎯─◐◑◒◓─ SHADERGLASS
Description: Applies .slang shaders in real time. **not compatible with Lossless Scaling.**
→ Launch ShaderGlass → Input → Window → select your game.  
→ Input → Pixel Size → x1
→ Shader → Choose From Library / Import Custom → choose shader. 
→ Output → Scale →  Scaling % **for custom scaling select FREE and use SRWE to adjust Shaderglass window**
   • Press 'm' to toggle menu on/off, useful when scaling with SRWE.
→ Performance tweaks **Undo if tearing presents in game**
   • Processing → Advanced → Use Flip Mode: Enabled; Allow Tearing: Enabled; Max Capture Rate: Enabled
   • Processing → FPS → 100% (VSync)
→ Shader suggestions:
   • RetroArch Library/pixel-art-scaling/sharp-bilinear **purity and edge integrity**
   • misc/simple_color_control **faithfully enhanced color depth**

⎯(ᶜᵒʷ•ᵔ)⎯─◐◑◒◓─ SRWE (SIMPLE RUNTIME WINDOW EDITOR)
Description: Tool for resizing, repositioning, and border‑taming stubborn apps and games.
→ Launch SRWE → Select Running Application → Shaderglass.exe **if not using Shaderglass, apply directly to 86Box vm window**
→ Adjust Window position and size → Set **maintain 4x3 aspect ratio to avoid stretching** **Press ENTER after entering values or they won't apply**
   • Hide title bar → Set Y: -31 **moves window title bar up and off screen**
   • Hide menu bar in Shaderglass → Processing → Toggle Menu (press 'm')
   • Accomodate for window frame: Width + 16, Height +39 **add to desired 4x3 resolution to fit and cover screen top to bottom**

⎯(ᶜᵒʷ•ᵔ)⎯─◐◑◒◓─ Coolsoft VirtualMIDISynth
Description: Host-side MIDI synthesizer with customizable soundfonts.
→ Launch VirtualMIDISynth → Add/Select soundfont → Apply
→ Suggested soundfonts:
   • For elevated accuracy: Arachno SoundFont - Version 1.0.sf2
   • For dreamy and atmospheric: GeneralUser GS v1.471.sf2
   • For SC‑55‑like correctness.: Unison.SF2

⎯(ᶜᵒʷ•ᵔ)⎯─◐◑◒◓─ RTSS (RivaTuner Statistics Server)
Description: Optional tool for an external framerate cap when using the Vulkan renderer.
→ Launch RTSS before 86Box **Launching after 86Box causes a crash**
→ Add → 86Box.exe
→ Framerate limit = 60 **favor compatibility**
